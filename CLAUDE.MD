## Architecture

- Single container: Fastify serves `/api/*` + React SPA
- Privileged mode: not required
- Job orchestration: dependency graph ‚Üí sequential execution ‚Üí retry logic
- NAS management: WOL ‚Üí health check ‚Üí backup ‚Üí graceful shutdown
- Backup types: Flash Drive, Appdata, VMs, Docker Images, System Config
- Multi-target: Remote server/NAS (SSH+Rsync), Local filesystem

## Unraid Integration

**Mounted volumes (read-only):**
/boot ‚Üí /unraid/boot              # Flash Drive (CRITICAL!)
/mnt/user ‚Üí /unraid/user          # User array
/var/run/docker.sock              # Docker API
/etc/libvirt ‚Üí /unraid/libvirt    # VM configs

**Pre-flight checks:** Array online (mdState=STARTED), Parity check not running, Mover not running, Target disk space sufficient

## Code Rules

**Frontend:**
- API calls ONLY via `api.ts` ‚Äî never `fetch` in components
- State mutations via Zustand ‚Äî never `api.*` directly in components
- Job Builder: visual dependency graph with @dnd-kit/core
- Live logs: SSE stream via `EventSource`
- Errors ‚Üí local state ‚Üí inline display

**Backend:**
- Interface for every request body and DB row (`CreateJobBody`, `JobRow`)
- Return `RowType | undefined` from `.get()` ‚Äî never `unknown`
- Errors: `reply.status(N).send({ error: '...' })` ‚Äî codes: 400/404/500/503
- Job execution: validate deps ‚Üí execute sequential ‚Üí log everything ‚Üí retry on failure
- SSH: reuse connections within job ‚Äî close after ‚Äî timeout 30s
- Credentials: sanitize responses ‚Äî strip `ssh_key`, `api_key`, `password`

## Database

Schema: Jobs (id, name, enabled, schedule, steps JSON), Targets (id, name, type, config JSON, enabled), JobHistory, Logs, Settings
- **Booleans**: SQLite `0 | 1` ‚Äî write `? 1 : 0`, read `=== 1`
- **JSON fields**: `JSON.stringify()` on write, `JSON.parse()` with fallback on read

## Job Structure
```typescript
interface Job {
  id: string
  name: string
  enabled: boolean
  schedule: string  // Cron
  steps: JobStep[]
}

interface JobStep {
  id: string
  type: 'wake_nas' | 'shutdown_nas' | 'rsync' | 'ssh_command' | 'docker_stop' | 'docker_start'
  depends_on?: string[]  // Step IDs
  config: Record<string, unknown>
  retry: { max_attempts: number; backoff: 'linear' | 'exponential' }
}
```

**Execution:** Build dependency graph (topological sort) ‚Üí execute in order ‚Üí retry on failure ‚Üí log everything ‚Üí update history

## Backup Types

**Flash Drive (`/boot`):** Rsync to target, exclude `previous/` and `System Volume Information/`, verify SHA256, export config JSON

**Appdata (`/mnt/user/appdata`):** Optional container stop ‚Üí Rsync (exclude `*/logs/*`, `*/cache/*`, `*/*.log`) ‚Üí export Docker configs ‚Üí restart

**VMs (`/mnt/user/domains`):** Libvirt snapshot if running ‚Üí Rsync vDisks ‚Üí export XML ‚Üí delete snapshot

**Docker Images:** List via API ‚Üí `docker save` to .tar ‚Üí store in target

**System Config:** Export JSON: network, users, shares, plugins, disk assignments

## Critical Gotchas

- **Node.js 24**: Use in Dockerfile `FROM node:24-alpine` and package.json `"engines": { "node": ">=24.0.0" }`
- **better-sqlite3 booleans**: write `? 1 : 0`, read `=== 1`
- **SSH key permissions**: must be 600 ‚Äî validate before use
- **Docker socket**: `undici.Pool` with `socketPath: '/var/run/docker.sock'`
- **Privileged container**: not required
- **Unraid array**: check `mdState` before backup ‚Äî never if array stopped
- **Cron validation**: use `cron-parser` library
- **Rsync bandwidth**: use `--bwlimit=50M` to avoid saturation
- **JSON in SQLite**: always `JSON.stringify()` / `JSON.parse()` with try-catch
- **pino-pretty**: must be in `dependencies` (not devDependencies)
- **SSH keys**: store in `/app/config/ssh/` ‚Äî never log private keys
- **Credentials**: sanitize all API responses ‚Äî strip passwords, tokens, keys
